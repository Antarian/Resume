Knowledge requirements for this tutorial:

  • Docker - installed
  • PHP - basics
  • Git/GitHub - basics

If you using some big framework (Symfony 4, Zend 2, CakePHP etc.) and place your code on GitHub to show some of your work, it is great. It’s good for others to see how you write the code, documentation and handle architecture or solve problems.

But large frameworks can be complicated for some small working example. Because people on other side must use same framework too and install it somehow, to see your code in action. Some of the frameworks can have really complicated setup and install.

This inspired me to create some small custom mini framework in 2016. And also to learn more behind basics of large frameworks.

Minitaurus FW must have basics

  • PHP5 >= 5.3.0 (namespaces)
  • configuration, environments
  • modules and layered (MVC) architecture
  • autoloading of classes
  • routing, with 404 page
  • debug
  • logs

Minitaurus FW nice to have attributes

  • template system
  • simple ACL
  • database access

I hope you have some environment, like Docker, Vagrant, LAMP, WAMP to develop in. I am using Docker on Ubuntu linux or on MacOS. It’s not in scope of this tutorial to setup different PHP environments. I will just place here my Docker setup. Dockerfile content.

# development docker file
FROM php:7.2-cli

LABEL minitaurus-dockerfile=Minitaurus

# prepare php ini settings
RUN mkdir /usr/src/php
RUN tar --file /usr/src/php.tar.xz --extract --strip-components=1 --directory /usr/src/php
RUN cp /usr/src/php/php.ini-development /usr/local/etc/php/php.ini

# prepare workdir
RUN mkdir -p /usr/src/minitaurus;
WORKDIR /usr/src/minitaurus

and also docker-compose.yaml file. May be little overkill for one docker file, but I am usually using this to centralise setup of multiple development machines.

# development docker-compose file
version: '3.6'
services:
  minitaurus:
    build:
      context: ./
      dockerfile: ./Dockerfile
    tty: true
    ports:
      - 8080:8080
    volumes:
      - type: bind
        source: ./
        target: /usr/src/minitaurus
    command: "php -S 0.0.0.0:8080"

Basic setup

We will change file index.php and define some constants for whole application.

<?php
/* index.php */
// define location of index.php as BASEDIR constant for easier manipulation with files
defined('BASEDIR') || define("BASEDIR", dirname(__FILE__));

// shortening constant from Directories - PHP file system related extensions included in core PHP
defined('DIRSEP') || define("DIRSEP", DIRECTORY_SEPARATOR);

We will need also one function which will make our life with PHP namespaces much easier.

// autoload classes through autoload function
spl_autoload_register(function($class) {{"{"}}
    // convert namespace to full file path
    $class = BASEDIR . DIRSEP . str_replace('\\', DIRSEP, $class) . '.php';
    try {{"{"}}
        // check if file exist
        if (!file_exists($class)) {{"{"}}
            throw new RuntimeException($class . ' file does not exist.');
        } else {{"{"}}
            require_once($class);
        }
    } catch(\Exception $e) {{"{"}}
        // handle the exception and stack trace
        echo nl2br('Error message: ' . PHP_EOL . $e->getMessage() . PHP_EOL);
        echo nl2br('Stack trace:' . PHP_EOL . $e->getTraceAsString() . PHP_EOL);
    }
});

This function is class auto-loader. It converts class name with namespace, one to one, to folder structure with class named file.

We can now add our first non-existent class as last line in index.php
